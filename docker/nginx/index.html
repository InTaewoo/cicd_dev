✅ 진행 과정

  • Terraform을 이용한 AWS EC2 기반 Kubernetes 클러스터(K8s) 구축
  • GitHub Actions를 활용한 CI/CD 파이프라인 설계
  • 코드 변경 시 애플리케이션 자동 빌드 및 테스트 환경 구성
  • Docker 컨테이너 이미지 빌드 및 AWS ECR에 저장
  • Kubernetes 환경에서 Kustomize 및 ArgoCD를 이용한 자동 배포 구성

✅ Trouble Shooting

  • 수정한 코드를 push할 때 무한 반복되는 현상 발생

→ checkout 단계를 추가하여 새로 생성되는 Repository가 아닌 지정된 Repository로 push되도록 설정

  • 이미지 Pull 권한 오류

→ EKS가 아닌 일반 Kubenetes로 구축하여 ImagePullSecret이 필요했고, Secret을 생성하여 정상적으로 이미지 Pull 확인

  • 새로 빌드한 이미지로 서비스가 정상 실행이 안되는 현상

→  Argocd Application을 생성할 때 사용한 namespace와 deployment namespace가 서로 달랐고 일치 시켜준 후 정상 실행

✅ 성과

  • IaC를 통한 클라우드 인프라 자동화 역량 습득
  • GitHub Actions 기반의 코드 커밋 → 빌드 → 배포 전체 파이프라인을 직접 설계 및 구현하며 CI/CD 환경 구축 경험 확보
  • Kubernetes 운영과 Kustomize, ArgoCD 실무 활용 능력 향상

  • AI 플랫폼을 기반으로 한 Agent를 자사 통계 솔루션에 연동하여, 사용자의 데이터 분석 및 활용 편의성을 높이기 위한 프로젝트를 기획하고 수행

✅ 진행 과정

  • Bare metal 서버 초기 환경 설정
  • Openshift 설치 및 플랫폼 설치
  • GPU 노드를 추가하여 생성형 AI 모델 배포
  • 통계 솔루션에 적용할 Agent 모델 개발
  • AI 플랫폼과 개발자가 개발한 Agent를 연동

✅ 담당 업무

  • 온프레미스 인프라 설계 및 초기 환경 설정
  • OpenShift 기반 AI 플랫폼 설치
  • GPU 노드 추가 및 MIG(Multi Instance GPU) 기능 구성
  • 플랫폼에 생성형 AI 모델 구축 및 개발된 Agent와의 연동

✅ Trouble Shooting

  • 보유한 GPU는 1장이었으나, 플랫폼 설치 및 AI 모델에 2장의 GPU가 필요

→ MIG(Multi Instance GPU) 기능을 활용해 GPU를 분할하여 용도에 맞게 사용

  • 50개 이상의 Pod가 Pending 상태로, 플랫폼이 정상적으로 실행되지 않음

→  Master/Worker 노드가 단일 노드로 구성되어 있었고, 노드당 최대 실행 가능한 Pod 수가 250개로 제한되어 있었음. kubelet 설정 파일에서 maxPods 값을 조정하여 정상 실행 확인

✅ 성과

  • Agent가 추가된 통계 솔루션의 판매량 30% 증가, 팀 내 최고 매출 달성
  • Agent 개발 이전 대비, 통계 솔루션 사용 고객의 만족도 90% 이상 증가
  • 월 2회 이상 고객사 방문 및 미팅을 통해 고객 유지율 상승
